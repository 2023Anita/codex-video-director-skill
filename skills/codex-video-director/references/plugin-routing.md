# Plugin Routing

Use the strongest tool for the shape of the video, not the tool that is most convenient.

## Figma

Use Figma when the task needs:

- moodboard exploration
- storyboard frames
- visual systems and token strips
- README cover direction
- stakeholder review before coding

Deliverable: a Figma-ready board or, when a Figma file is available, a written frame plan that can be translated into the file.

## HyperFrames

Use HyperFrames when the task needs:

- cinematic HTML motion
- title cards
- captions or voiceover timing
- GSAP transitions
- trailer-like visual rhythm
- lint, inspect, preview, render workflow

Deliverable: an HTML composition with explicit timing and a `DESIGN.md` or equivalent style contract.

## Remotion

Use Remotion when the task needs:

- React component video
- data-driven timelines
- charts, milestone cards, and reusable scenes
- parameterized exports
- batch generation from JSON

Deliverable: a composition driven by JSON props and reusable React components.

## Combined Route

For serious case studies:

1. Figma defines the visual system.
2. Remotion renders data-driven timeline sections.
3. HyperFrames handles cinematic openers, closers, transitions, and caption rhythm.
4. Codex assembles the repo, validates assets, and packages the skill.
